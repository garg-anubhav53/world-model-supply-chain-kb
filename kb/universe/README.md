# THE UNIVERSE — a coverage FRAMEWORK, not a checklist

**Purpose:** a holistic way for the routine + KB to verify that the *full ecosystem of viable public opportunities* has been parsed — by ensuring every **surface** (a cell across the dimensions below), reached via a rotation of **discovery sources**, is either COVERED or explicitly justified-blind. **Completeness is measured by DIMENSION coverage, not by name count.**

**This is deliberately loose and flexible.** The routine must NOT march name-by-name through a list. It must find the least-explored surface, *discover* names within it, and move on. The names in `seed-map.md` are **illustrative seeds/examples**, not an assignment — no one built this in 30 minutes could know every interesting surface, so the framework, not the list, is the guarantee.

## The dimensions of the ecosystem (the surfaces to cover)
- **D1 · NODES (supply-chain layers):** compute (GPU/HBM/networking/optical+subcomponents/servers/advanced-packaging); DC-physical (power/cooling/passives/storage); robot-mechanical (actuators/reducers/motors/laminations/ball-screws/bearings/structural/gear-machine-tools); robot-sensing+electronics (force-torque/tactile/encoders/IMU/battery-BMS/edge-compute/motor-drivers/MCU); magnets & rare-earth & specialty materials; perception/AV (lidar/radar/camera/depth/machine-vision/HD-map/auto-compute/sensor-cleaning); data/software (mocap/annotation/simulation/teleop/data-capture-rigs); semicap equipment. **PLUS structurally-added nodes:** consumables/aftermarket (per-operating-hour spend), interconnect (high-flex cable/connectors/slip-rings), positioning & timing (GNSS/RTK/oscillators/PTP), edge/onboard thermal, production-test/calibration/validation, semi & magnet materials-consumables (gases/targets/slurry/quartz), real-world data-capture hardware (egocentric/event cameras, wearable IMU, auto data-loggers). *Node list is OPEN — add nodes when a fire finds one.*
- **D2 · GEOGRAPHIES:** US · Japan (incl. TSE Growth/Standard) · Korea (KOSPI+KOSDAQ) · Taiwan (TWSE+TPEx) · **China (A-share/STAR/Shenzhen/Beijing + HK/ADR)** · Europe (multiple exchanges) · Israel · India · SE-Asia · Canada · Australia · LatAm.
- **D3 · BUSINESS MODELS:** component-BOM · consumable/aftermarket (recurring) · royalty/IP-licensing & patent pools · capacity-lease / EMS-ODM · TIC / certification · insurance / surety · recycling / circular · marketplace / data.
- **D4 · TRACKS & adjacent demand verticals:** W (world-model labs) · R (humanoid/robotics) · AV — PLUS adjacent embodied-AI verticals that pull the SAME forced inputs: drones/UAV/counter-UAV, defense autonomy, agri-robotics, surgical/medical robotics, maritime/port, mining/construction, warehouse/logistics, space.
- **D5 · DISCOVERY SOURCES/METHODS** *(rotate; each fire must use ≥1 NON-anchored source):* anchored = ETF/index holdings, 13F-delta, exchange sector screens. **NON-anchored (the edge, because they don't require prior index membership)** = buyer-BOM teardown · EDGAR "significant customer" · **customs / bill-of-lading import records** (highest-yield: who physically ships parts to Figure/Tesla/Waymo) · patent assignees (CPC B25J/F16H) · standards-body rosters (ISO TC299, SAE AV, AUTOSAR, PTP) · government tenders/procurement · buyer job-postings · earnings-call SUPPLIER mentions · native-language trade press (集微网/高工机器人, JP IR PDFs, KR DART) · forward-listing pipeline (S-1/F-1, spin-offs, de-SPAC, pivots).
- **D6 · SIZE/LIQUIDITY:** deliberately fish BELOW the liquidity line (nano/micro) — that's where un-re-rated multipliers hide (SOS Lab, $4.2M base, proved it).
- **D7 · TEMPORAL frontier:** not-yet-listed names (IPO/spin/de-SPAC/pivot) — un-re-rated by definition.

## Standing ANTI-BIAS rules (keep the universe widening, never calcifying)
1. **Every fire touches ≥1 NON-anchored D5 source** — a fire that only ran index/keyword screens is incomplete.
2. **Consumables-first reframe** on every node: ask "what's bought per operating-hour, not per unit?" — recurring-spend beneficiaries rank above one-time-BOM at equal causal strength.
3. **Business-model rotation:** advance the least-covered D3 each fire (today only component-BOM is deeply worked).
4. **Geography anti-skew:** China A/STAR is ACTIVE (Connect/HK/ADR-accessible), not "map-only"; don't end 2 fires in a row with India+SE-Asia+Israel all blind.
5. **Mandatory survivor back-propagation:** every new WATCH auto-spawns "who supplies THIS name?" + "any IPO/spin-out into this node?" next fire.
6. **Forward-listing watch (D7):** periodic scan of new listings / spin-offs / de-SPACs / robotics pivots.
7. **Dead-ends decay:** quarterly re-test every "no public vehicle" entry (a pending IPO flips private→listed).
8. **Bias-ledger:** each fire logs which dimensions it reduced (`kb/coverage-ledger.md`); any dimension idle 3 fires → forced next.
9. **Names are seeds, not a checklist:** discover *within* surfaces; never iterate a fixed list.

## How each fire uses this
Read `coverage-ledger.md` → pick the least-explored surface (a node×geo×business-model reachable via an under-used discovery source) → discover + gate + diligence names there → update the ledger + pipeline. The point is to prove the *space* is covered, flexibly — not to tick off names.
